# 02 -- Stateful and Metamorphic Testing

## Backstory

Somewhere in the scraper pipeline's hot path sits a small in-memory cache:
fetch a page, check the cache first, only hit the network on a miss, and
expire entries after a while so the cache does not serve stale content
forever. It is a tiny piece of code -- an `OrderedDict` and some
bookkeeping -- and that is exactly why bugs in it are dangerous: it is
"obviously correct" until a request pattern hits the one sequence of
operations nobody happened to write a test for. A cache with a
recency bug does not crash. It just quietly evicts the wrong entry, and
the pipeline re-fetches a page it didn't need to, or serves a
just-expired value one request too many. Nothing raises. Nothing shows up
in a log. It just costs money and correctness, slowly, forever.

This task hands you a correct `TTLCache` (`src/impl.py`) and asks you to
write a test suite good enough that you would trust it to catch a
regression in this class before it ships -- specifically, one written by
someone who "cleaned up" the eviction logic, or "simplified" the TTL
check, without meaning to change behavior.

## What's given

- `src/impl.py` -- a correct `TTLCache` with an injectable clock. Read the
  module docstring closely: it pins down the exact eviction rule, the
  exact recency rule (what does and does not refresh a key's position),
  and the exact TTL boundary (`clock() >= deadline` is expired -- the
  deadline instant itself already counts). Do not edit this file; it is
  the given, correct implementation, not the deliverable.
- `src/sut.py` -- a generated shim. Your tests import `TTLCache` from
  here (`from src.sut import TTLCache`), never from `src.impl` directly.
- `tests/test_cache.py` -- a scaffold with guidance and TODOs, no active
  tests. This is the file you fill in; it's the actual deliverable.
- `tests/validate.py` -- the grader. Do not edit it.
- `.authoring/` (at the module root) -- off-limits. It documents the
  hidden mutant bank your suite is graded against. Do not read it before
  finishing this task; it is the answer key.

## What's required

Write `tests/test_cache.py` so that it:

1. Fully passes against the given, correct `TTLCache`.
2. Fails against every mutant in the hidden bank -- each mutant is a full
   copy of `TTLCache` with exactly one bug (eviction order, TTL boundary,
   capacity, or recency handling).

You need two kinds of tests, and neither alone is enough:

- A Hypothesis **stateful** test
  (`hypothesis.stateful.RuleBasedStateMachine`) that drives a sequence of
  `put`/`get`/advance-the-clock operations against both the real cache
  and a small reference model, asserting they agree after every step.
  This is what catches bugs that only show up after a specific SEQUENCE
  of operations -- eviction order and recency bugs are exactly this
  shape; no single call in isolation reveals them.
- A handful of **metamorphic** properties -- relations that must hold
  regardless of the specific input, e.g. "a `get` immediately after a
  `put` of the same key returns that value," "`len(cache)` never exceeds
  `capacity`," "a key is never observably alive at or past its TTL
  deadline."

Drive time with the injected `clock` parameter -- construct the cache
with a controllable fake clock (a mutable box you can reassign) and
advance it by reassignment. Never call `time.sleep()` in a test; it makes
tests slow and still non-deterministic at the boundary you actually care
about.

## Completion criteria

From the module root:

```bash
uv run python 02-stateful-and-metamorphic/tests/validate.py
```

Prints `PASSED` (with a `killed N/N mutants` detail line) when your suite
passes against the correct implementation, collects at least the minimum
number of tests, and fails against every mutant. Prints
`NOT PASSED: <reason>` and exits 1 otherwise -- including while
`tests/test_cache.py` is still just the scaffold (0 tests collected is a
clean failure, not a crash).

## Topics to read up on

- Stateful / model-based testing: what problem it solves that plain
  property-based testing does not
- Hypothesis's `RuleBasedStateMachine`, `@rule`, `@invariant`, and
  `Bundle`
- Metamorphic testing: asserting a RELATION between inputs/outputs rather
  than a single expected output
- Deterministic time in tests via an injected clock, instead of
  `time.sleep()` and real wall-clock waits
- LRU eviction and TTL expiry semantics, and the boundary conditions that
  tend to hide off-by-one bugs in both
