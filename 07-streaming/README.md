# 07 - Streaming

Kafka (via redpanda) framed specifically for someone who already thinks in RabbitMQ terms: the log-vs-queue distinction, consumer offsets and rebalancing, exactly-once semantics, and compacted topics as a different way to model state. The goal is to make the mental model differences from AMQP-style messaging explicit, not just teach a new API.

Includes an optional k8s-bonus level: running the streaming stack on a local kind/k3d cluster with your own Helm chart.
