# Hint 1

Write every section as a rule you could actually enforce in a PR review
or a branch-protection setting, not as a value you hold. "Commits should
be meaningful" is a value. "A commit must leave the test suite passing"
is a rule -- someone can check it. Go through each section of the
template and ask yourself: if a teammate broke this, what specific thing
would you point at in review?

The hostile-review questions are not a separate creative-writing
exercise tacked on at the end -- each one is designed to hit a case your
policy sections above should already resolve. Write the sections first
with those six scenarios in mind, and the answers become much shorter to
write.
