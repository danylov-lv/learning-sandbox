`prepare_events`: `spark.read.json(str(jsonl_dir / "*.jsonl"))`, then `.filter(F.col("_corrupt_record").isNull()).drop("_corrupt_record")`, then `.distinct()`, then `.select("product_id", "source_id", "price", "http_status", "captured_at")`. No status filter here — that's each downstream function's own job.

`top_n_per_source`: filter to `http_status == 200` first. Build one `Window.partitionBy("source_id").orderBy(F.desc("price"), F.desc("product_id"))` and apply `F.row_number()` over it in a `withColumn`, then `.filter(col("rn") <= n)`, then select down to `source_id, product_id, price, rn`. That's the entire function — the work is in choosing `row_number()` over `rank()`/`dense_rank()` (see hint-2) and getting the orderBy's two sort keys in the right order and direction, not in any extra logic beyond that.

`price_change_per_product`: you need two different orderings of the same partition, so two `Window.partitionBy("product_id")` specs — one `.orderBy("captured_at")` ascending (feed this to `F.lag("price")` and `F.lag("captured_at")` to get each row its predecessor's values), one `.orderBy(F.desc("captured_at"))` (feed this to `F.row_number()` and filter to `1` to isolate each product's most recent row *after* the lag columns have already been computed on the full per-product sequence — compute the lag first, filter to the latest row second, not the other way around, or the lag will see the wrong "previous" row). The final `price_delta` column is just `price - prev_price` computed with plain arithmetic on two already-materialized columns, and rename `price`/`captured_at` to `latest_price`/`latest_captured_at` in your final `select`.

`window_vs_aggregate_plans`: the windowed branch is literally `top_n_per_source`'s logic with `n=1` inlined (same partitionBy/orderBy, `row_number() == 1` instead of `<= n`) — you can build it directly rather than calling your other function, since the validator only checks the plan text and the resulting max-price-per-source values, not which function produced them. The aggregate branch is `valid.groupBy("source_id").agg(F.max("price"))` — nothing more. Capture each with `get_plan(...)` (works fine on an unmaterialized DataFrame), and turn each result into a `{source_id: price}` dict with `.collect()`, keying with `str(row["source_id"])` to match `ground-truth.json`'s string keys.
