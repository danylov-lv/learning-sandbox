**`with_python_udf`**: define a plain Python function taking `(price, attrs)` and returning a `(weight_g, bucket)` tuple. `attrs` arrives as a `pyspark.sql.Row` (or `None`) тАФ accessing a missing/null field on it, or a `weight_g` that isn't numeric, should not raise; guard it and fall back to `None`. Register the function with `F.udf(fn, schema)` where `schema` is a `StructType` with fields `weight_g: DoubleType` and `bucket: StringType`, apply it with a single `withColumn` producing a struct column, then pull the two fields back out into top-level columns with `struct_col["weight_g"]` / `struct_col["bucket"]`.

**`with_pandas_udf`**: a struct-returning `pandas_udf` wants a function that takes one or more `pandas.Series` arguments and returns either a `pandas.DataFrame` (columns matching your struct fields) or a `pandas.Series` of dicts, depending on how you declare the schema тАФ a DataFrame return with `@F.pandas_udf(struct_schema)` is the straightforward path. Extract `attrs.weight_g` as a JVM-side double column *before* the UDF call (a plain `withColumn`, not part of the pandas function), and pass `price` and that pre-extracted column in as the UDF's two Series arguments тАФ this keeps struct-drilling out of the vectorized function, which only needs to do numeric/string work.

**`with_builtins`**: `F.col("attrs")["weight_g"].cast("double")` gets you `weight_g` directly тАФ no UDF involved, struct field access is a native Catalyst expression. For `bucket`, chain `F.when(cond1, "a").when(cond2, "b").otherwise("c")` тАФ remember `when` conditions are checked top to bottom and the first match wins, so order the null check first.
