Everything you need for `load_events`, `monthly_rollup`, `filter_probe`, and `top3_per_source` is a job shape you've already built once in Spark, in tasks 01, 01/02, 01, and 05 respectively — same filters, same grouping keys, same tie-break rule. The only genuinely new problem here is getting clean data into polars in the first place: the raw dump has lines that aren't valid JSON at all, and the NDJSON reader's error-tolerance option doesn't do what its name suggests. Read the module docstring in `src/calibrate.py` before you try anything — it tells you what was actually verified and what wasn't.

Stay lazy. Every one of these four functions can be built and returned without ever calling `.collect()` inside it (except where you need a scalar out — a row count or a sum — to put in a returned dict). Chaining `.lazy()` around something you already collected defeats the entire premise of the comparison you're about to run in `tests/bench.py`.
