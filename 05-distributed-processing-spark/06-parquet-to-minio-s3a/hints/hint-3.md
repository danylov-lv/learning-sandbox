For `write_month_partitioned`: read `jsonl_dir`'s `*.jsonl` files, `.filter(col("_corrupt_record").isNull())`, `.drop("_corrupt_record")`, `.dropDuplicates()` (whole-row — no column list argument), `.withColumn("month", F.substring(F.col("captured_at"), 1, 7))`, then `.repartition("month").write.mode("overwrite").partitionBy("month").parquet(dest)`. Get your per-month counts and total either from the DataFrame before the write (cache it first if you count before and after) or by reading `dest` back afterward and `.groupBy("month").count()` — reading back is simpler to reason about since it's exactly what the lake actually contains once written.

For `inspect_lake_files`: `spark.read.parquet(dest)`, add a column with `F.input_file_name()`, `.select("month", <that column>).distinct().collect()` to the driver (18 months × 1 file = 18 rows if correct, so this is small regardless of dataset scale), then group by month in plain Python to build `files_per_month`, and check that each row's file-name string contains the substring `f"month={row['month']}"`.

For `pruned_read`: `df = spark.read.parquet(dest).filter(F.col("month") == month)`; capture `get_plan(df, "formatted")` immediately (before `.count()` or anything else runs it); then `row_count = df.count()`; then `distinct_files = df.select(F.input_file_name().alias("f")).distinct().collect()`, `len(distinct_files)` is your `distinct_files_touched`. The field to check in the captured plan text is `PartitionFilters:` on the `Scan parquet` node — verified empirically against pyspark 3.5.3 reading a `partitionBy`-written table on s3a: a month-equality filter produces a non-empty `PartitionFilters` list that includes the literal month value being filtered on, while a table scan with no partition predicate at all shows `PartitionFilters: []`. `PushedFilters` on the same node is a separate, unrelated mechanism (row-level predicate pushdown within a file). In `formatted` mode it doesn't even appear on this scan node — verified empirically: with the whole predicate satisfiable from the partition directory name alone, there's nothing left to push into the Parquet reader, and the field is simply omitted rather than printed empty (compare against `simple` mode, where the same scan prints `PushedFilters: []` explicitly).
