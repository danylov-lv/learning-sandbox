For `job_counts_around_actions`, the shape is: read → record count A → `.filter(...).select(...)` (no action) → record count B (should equal A) → `.count()` → record count C (should be A/B + 1) → `.collect()` on the same filtered/selected DataFrame → record count D (should be C + 1). Every "record count" call is just `len(spark.sparkContext.statusTracker().getJobIdsForGroup())`.

For `narrow_vs_wide_plans`, one function, two DataFrames, one shared setup (read the raw JSONL, drop `_corrupt_record` rows). The narrow one stops at `.filter(...).select(...)`. The wide one adds `.groupBy("source_id").agg(F.count("*"))` or similar. Get each plan with `get_plan(df)` before you ever call an action on either — you don't need to run them to read their physical plan.

For `pushdown_comparison`, remember `spark.sql.adaptive.enabled` must be `false` for the plan you inspect to reflect what will actually execute rather than a placeholder `AdaptiveSparkPlan(isFinalPlan=false)` shell — set it once at the top of the function. Build the same `.filter(col("source_id") == 4) & (col("captured_at") >= ...) & (col("captured_at") < ...)).select("source_id", "price")` query against the JSONL-read DataFrame and against the Parquet-read DataFrame, and capture both plans. Do not call `.distinct()` in this function — dedup and pushdown are separate concerns here (see the README's completion criteria for why: whole-row dedup needs every column, which defeats projection pruning below it).

For `bootstrap_parquet_slice`, this is a straight read-filter-write: read all of `jsonl_dir`'s `*.jsonl`, `.filter(col("_corrupt_record").isNull())`, `.drop("_corrupt_record")`, `.write.mode("overwrite").parquet(str(out_dir))`. Count the rows before or after the write (`df.count()` before writing avoids a second full scan if you assign the filtered DataFrame to a variable first — either is correct, one just costs more).

For `dedup_filter_probe`, this is the same read-and-filter shape as above, plus `.distinct()` before your date/source filter (distinct on the whole row — do not pre-select columns first), plus the source/date predicate, then `.count()` for rows and a conditional `.filter(col("http_status") == 200).agg(F.sum("price"))` for the price sum. The date bounds: `captured_at` is stored as an ISO-8601 string, so string comparison against `"2025-09-01"` and (half-open) `"2025-11-01"` works correctly for this format.
